Responsive Images Problem

2 major use cases

Let's better define what we're talking about when we're talking about the "responsive images problem"

Resolution switching

Serving different image dimensions to different devices

Image resources only differ in quality. proportions stay the same

DPR switching

Sending retina images only to retina screens

viewport switching

Sending images adapted to their display dimensions
For both res switching, browser should be able to serve lower res resources

Art Direction

Getting images to match layout

Fit layout and optimal content without wasting bytes
40% of devs want art direction

Other use cases

  • Monochrome
  • Print
  • Image format fallback

Mobify.js

A whole new level of "Bat-sh*t-loco-insane" hack

What?

Provides script access to the HTML document before the preloder or parser gets to see it JS can rewrite image URLs to adapt them to viewport and DPR

How?????

  • Inline script at the doc head injects a <plaintext> tag and downloads mobify.js
  • Browser treats the rest of the document as text
  • HTML chunks are read &alt; processed as they're downloaded
  • Once HTML is done, document.open() and document.write() the new HTML
  • Page load starts over, preloader and all

Advantages

Very easy to convert legacy convert

Disadvantages

Blocking JS at the start + Waiting for the entire HTML to download Lots of JS, not sure legacy phones would appreciate

Be careful of polyfill that use src. You will hit a double download.
